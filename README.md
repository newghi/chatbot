# 챗봇 구축 전략

본 프로젝트는 초기 서비스 안정성과 빠른 도입을 위해 벨루가(Veluga) AI 챗봇 API를 활용한 UI 커스터마이징 방식으로 시작하며, 이후 서비스 운영 데이터와 고객 질문 로그를 기반으로 자체 AI 챗봇 시스템으로 단계적으로 전환하는 전략을 적용한다.

이 방식은 초기 개발 비용과 운영 리스크를 최소화하면서도 장기적으로는 AI 데이터 자산 확보 및 운영 비용 절감을 동시에 달성할 수 있다.

## 1. 전략 개요

초기에는 외부 AI 엔진을 활용해 빠르게 서비스를 구축하고, 운영 데이터를 축적한 뒤 RAG 기반 자체 챗봇으로 전환하여 확장성과 비용 효율성을 확보한다.

## 2. 1단계: Veluga API 기반 챗봇 구축 (초기 단계)

초기 단계에서는 벨루가의 AI 엔진을 API 형태로 활용하고, 챗봇 UI와 사용자 경험(UX)은 자체적으로 개발하여 서비스에 적용한다.

### 주요 특징

- Veluga AI 엔진 활용
- 자체 챗봇 UI 개발
- API 기반 데이터 통신
- 빠른 서비스 도입

### 시스템 구조

```text
사용자
 ↓
웹사이트 챗봇 UI
 ↓
API 서버
 ↓
Veluga API
 ↓
AI 응답
```

### 주요 구현 기능

#### 2.1 UI 커스터마이징

벨루가 기본 위젯 대신 자체 채팅 UI를 개발하여 서비스 디자인과 일관성을 유지한다.

- 채팅 인터페이스 디자인
- 브랜드 컬러 적용
- 사용자 맞춤 UI
- 모바일 최적화

#### 2.2 대화 로그 수집

벨루가 API 응답과 별도로 모든 대화 로그를 자체 DB에 저장한다.

- 사용자 질문
- AI 응답
- 응답 시간
- 사용자 피드백

이 데이터는 향후 자체 챗봇 학습 데이터로 활용된다.

#### 2.3 트래픽 비용 관리

벨루가 플랫폼은 API 사용량 및 트래픽 정책에 따라 비용이 발생하므로 다음과 같은 관리 전략을 적용한다.

- 요청 수 제한
- 캐싱 전략 적용
- FAQ 우선 응답
- 트래픽 모니터링

## 3. 2단계: 데이터 기반 챗봇 개선

초기 운영 단계에서는 챗봇 서비스 데이터를 지속적으로 수집하고 분석하여 지식 데이터베이스(Knowledge Base)를 구축한다.

### 주요 활동

- 고객 질문 데이터 분석
- FAQ 및 지식 데이터 구축
- 관리자 운영 시스템 구축

**수집 데이터**

- 고객 질문 유형
- AI 응답 정확도
- 미해결 질문

**Knowledge Base 구성 문서**

- 서비스 정책
- 제품 정보
- 이용 가이드

**관리자 기능**

- 미해결 질문 관리
- FAQ 추가
- 지식 문서 업데이트
- 챗봇 성능 분석

## 4. 3단계: 자체 AI 챗봇 시스템 구축

서비스 데이터가 충분히 축적되면 벨루가 의존도를 줄이고 자체 AI 챗봇 시스템으로 전환한다. 이 단계에서는 RAG 기반 AI 챗봇 구조를 적용한다.

### 시스템 구조

```text
사용자
 ↓
웹사이트 챗봇 UI
 ↓
API 서버
 ↓
Vector Search
 ↓
LLM API (OpenAI / Gemini)
 ↓
AI 응답
```

### 주요 기술 구성

- AI 엔진: OpenAI GPT API, Google Gemini API
- RAG 시스템: Vector DB 기반 문서 검색 후 AI 응답 생성
- 기술: LangChain, LlamaIndex
- 데이터베이스: PostgreSQL, pgvector (Vector DB)
- 데이터 수집 시스템: 웹사이트 URL 크롤링, 파일 업로드 기반 문서 학습, 관리자 입력 기반 지식 추가

## 5. 4단계: AI 서비스 고도화

자체 챗봇 시스템 구축 이후에는 고객 맞춤형 AI 서비스로 확장한다.

### 확장 기능

- 고객 맞춤 응답 (회원 데이터 기반 응답: 주문 상태 안내, 개인 맞춤 추천)
- 서비스 이용 안내 자동화
- 자동 마케팅 지원 (고객 관심 분석, 인기 상품 분석, FAQ 자동 생성)
- 상담 자동화 및 상담원 연결 시스템 구축

## 6. 단계별 구축 로드맵

| 단계 | 내용 |
| --- | --- |
| 1단계 | Veluga API 기반 챗봇 구축 |
| 2단계 | 챗봇 데이터 수집 및 Knowledge Base 구축 |
| 3단계 | RAG 기반 자체 챗봇 시스템 개발 |
| 4단계 | 고객 맞춤형 AI 서비스 확장 |

## 7. 기대 효과

**초기 단계**

- 빠른 서비스 도입
- 개발 비용 절감
- 안정적인 AI 서비스 운영

**장기 단계**

- AI 데이터 자산 확보
- 운영 비용 절감
- 서비스 커스터마이징 강화

## 8. 결론

본 전략은 초기에는 Veluga API 기반 챗봇 서비스를 활용하여 빠르게 서비스를 도입하고, 이후 서비스 운영 데이터를 기반으로 자체 AI 챗봇 플랫폼으로 단계적으로 전환하는 방식이다.

이를 통해 초기 개발 리스크 최소화, AI 데이터 자산 확보, 장기적인 운영 비용 절감을 동시에 달성할 수 있다.
